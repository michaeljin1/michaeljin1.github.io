MECE 멀티 에이전트 개발 환경

페로몬 프로토콜에서 착안. md 공유 + 훅 트리거로 자율 실행.

Claude Code Sub-agents Hook 자동화 권한 격리 (MECE)
planner writesspec.md →onChange
builder reads spec.md, writesimpl/*.ts →onChange
reviewer reads impl/*, writesreview.md →done
hook · 변경 감지 시 다음 에이전트만 호출 — 메인 컨텍스트 오염 0, 단일 명령으로 자율 실행, 로그 누적·재시작 가능.
기존 방식의 문제
Claude Code 내 슬래시 명령으로 sub agent (기획·디자인·리뷰)를 호출했지만, 메인 컨텍스트 오염, 명령 번거로움, 데이터/로그 휘발이 발생했습니다.
새 방식
페로몬 프로토콜에서 착안해 sub agent들이 md 파일을 공유하고, 변경 시 훅으로 다음 에이전트를 호출하도록 설계했습니다. MECE 권한 격리.
실험과 절충
1회 명령만으로 자율 실행, 단계별 산출물 품질 향상, 로그 누적·재시작 가능. 비용은 느린 실행, 횡단 검증 불가, 작업 과잉. 비용 인지 후 긴 호흡 작업에만 적용하고, 내용에 따라 필수 agent만 채택하도록 최적화했습니다.
