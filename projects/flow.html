MECE 멀티 에이전트 개발 환경

슬래시 명령으로 매번 sub-agent를 부르는 방식은 메인 컨텍스트가 빠르게 오염되고, 산출물·로그가 휘발됐다.

페로몬 프로토콜에서 착안해 에이전트 간 통신은 md 파일로, 변경 감지는 훅이 담당하도록 설계. 한 번 명령하면 단계가 자율 실행되고 모든 산출물이 파일로 누적된다.

Stack
Runtime
Claude Code Sub-agents
Trigger
PostToolUse Hook 파일 변경 감지
Storage
Markdown 파일 권한 격리 (MECE)

파이프라인

plannerwrites spec.md→onChange
builderreads spec.md, writes impl/*.ts→onChange
reviewerreads impl/*, writes review.md→done
hook · 변경 감지 시 다음 에이전트만 호출 — 메인 컨텍스트 오염 0, 단일 명령으로 자율 실행, 로그 누적·재시작 가능.

시스템 아키텍처

사용자 — 단일 명령
↓
planner
요구사항 → spec.md
builder
spec.md → impl/*
reviewer
impl/* → review.md
↓
PostToolUse Hook
파일 변경 감지
권한 매트릭스
read/write 격리
로그 누적
md 파일 = 진행 상태

한 사이클 처리 흐름

1
사용자가 단일 명령 입력 — planner 호출 ~0s
2
planner → spec.md 작성 (요구사항·제약·산출물 정의) ~30s
3
Hook 발동 → builder만 호출, spec.md 읽고 impl/* 작성 ~2–5min
4
Hook 발동 → reviewer만 호출, impl/* 읽고 review.md 작성 ~1–2min
5
사이클 종료 — 모든 산출물이 md 파일로 누적, 재시작 가능 ~0s

권한 격리 원칙

·
각 에이전트는 자기 산출물만 write — 교차 쓰기 금지 MECE write
·
상류 단계 산출물만 read — 하류 산출물 참조 금지 one-way read
·
에이전트 간 메모리 공유 없음 — 통신은 md 파일이 유일 file as bus
·
긴 호흡 작업에만 적용 — 짧은 작업은 단일 명령 직접 실행 cost-aware
